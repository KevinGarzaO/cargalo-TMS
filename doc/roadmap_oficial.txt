Criterio de priorizacion
Cada semana responde una sola pregunta: si el cliente tuviera acceso hoy, que necesitaria para generar valor? Lo que responde esa pregunta primero va primero.

Definition of Done por sprint
Una semana es DONE cuando el cliente puede usar lo prometido en el lanzamiento sin bugs criticos. No es done si solo funciona en local o si hay flujos rotos.
Criterio
Descripcion
Funciona en produccion
El feature esta en el ambiente real — no solo en local ni en staging
Sin bugs criticos
No hay errores que bloqueen el flujo principal del usuario
Roles respetados
Los permisos funcionan correctamente para todos los roles involucrados
Responsive
Para features del panel web: funciona bien en celular ademas de desktop
App en dispositivo real
Para features de la app: probado en al menos 2 Android fisicos distintos
Datos reales
Se probo con datos reales, no solo con datos de prueba inventados
Claude Code doc actualizado
Si hubo cambios de arquitectura o BD se actualizo el documento correspondiente
